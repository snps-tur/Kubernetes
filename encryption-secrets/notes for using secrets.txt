To create a secret using CLI:
kubectl create secret generic <my-secret-name> --from-literal=key1=supersecret

To describe a secret:
kubectl describe secret <my-secret-name>\

To describe in detail:
kubectl describe secret <my-secret-name> -o yaml


Note: Using CLI will auto encode the password, you don't manually have to encode it, only when using declarative method (creating a file) we have to encode manually

Command to encode the data manually:
echo -n '<plain text>' | bas64

Command to decode the data manually:
echo -n '<plain text>' | bas64 --decode
